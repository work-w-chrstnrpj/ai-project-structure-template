# Task Breakdown

## Use When

Break features into implementation tasks, milestones, dependencies, and owners.

## Workflow

1. Read `AGENTS.md` and follow project guardrails.
2. Use `.ai/context-routing.md`, compact maps, indexes, and search to find the smallest relevant context.
3. Read exact source files, tests, contracts, docs, or configs before making claims or edits.
4. Apply the planning concern narrowly and preserve existing project patterns.
5. Report outputs, verification needs, risks, and handoffs.

## Boundaries

- Do not treat generated maps, graphs, embeddings, summaries, or snapshots as implementation truth.
- Do not cross into another role's ownership without a clear handoff.
- Do not perform destructive, production, migration, secret, or high-risk actions without approval.

## Output

- Relevant context inspected.
- Decisions or changes made.
- Tests, checks, or verification needed.
- Risks, assumptions, and handoffs.
