# OpenCode Implementation Prompt: Tool Usage Policy for AI Project Structure Template

## Goal

Update this repository so Aider, Graphify, Repomix, Understand Anything, and Caveman are used correctly as optional, context-efficient workflow tools.

Do not refactor agents or skills in this task.
Do not expand the tool system.
Do not make every tool run for every prompt.
The goal is a generic, project-agnostic, language-agnostic, reusable AI-driven project structure.

Core principle:

```txt
Default workflow = routing docs + targeted search + exact source files + tests.
Tools = escalation only when they add value.
```

## Scope

Focus only on these tools:

1. Aider
2. Graphify
3. Repomix
4. Understand Anything
5. Caveman

Do not modify unrelated roles, personas, subagents, or skill catalogs unless a small link/reference update is required because of a renamed file.

---

## Desired Tool Hierarchy

Use this hierarchy consistently in the repository documentation:

```txt
Primary but not automatic:
- Aider: optional terminal coding executor
- Graphify: optional repository relationship/navigation map
- Repomix: optional portable context snapshot exporter

Optional:
- Understand Anything: optional onboarding / visual exploration / broad understanding tool
- Caveman: optional terse-output style mode
```

Important clarification:

```txt
Primary does not mean always invoked.
Primary means recommended when the workflow actually needs that capability.
```

---

## How context-strategy.md should work

`.agents/tools/context-strategy.md` must be treated as a routing guide, not a launcher.

It should explain this flow:

```txt
1. Read core project guidance.
2. Use context routing / maps / indexes if available.
3. Use targeted search such as rg or exact file lookup.
4. Invoke a tool only if the task needs that tool.
5. Read exact source files before editing.
6. Verify with tests or relevant checks.
```

It must clearly say:

```txt
Generated graphs, repo maps, snapshots, summaries, and dashboards are navigation aids only.
Exact source files and tests are the source of truth.
```

## Required file changes

### 1. Update or create `.agents/tools/context-strategy.md`

Make it concise and strict.

It should define this decision table:

```md
| Task situation | Default action | Tool escalation |
|---|---|---|
| Small known-file edit | Read exact file(s), edit, test | No tool |
| Unknown dependency/call path | Search first, then use relationship map if needed | Graphify |
| Coding inside terminal AI workflow | Use exact files and repo map | Aider |
| Need to send repo context to another AI/tool | Export small controlled snapshot | Repomix |
| Large unfamiliar repo / onboarding | Use broad exploration aid | Understand Anything |
| User asks for terse output / low-risk status | Use concise style | Caveman |
```

It should include this rule:

```md
Do not run Graphify, Repomix, Understand Anything, or Caveman for every prompt.
Do not use Repomix as default coding context.
Do not edit code based only on generated context.
```

# Aider

## Role

Aider is an optional terminal coding executor.

## Use when

### 3. Update `.agents/tools/graphify.md`

Content should say:

```md
# Graphify

## Role

Graphify is an optional repository relationship/navigation map.

## Use when

- exploring unfamiliar code areas
- finding dependencies or call paths
- planning refactors
- onboarding an agent to a module
- identifying likely files before exact reads

## Do not use when

- the target files are already known
- simple search is enough
- exact implementation behavior is needed
- the generated graph may be stale and no source verification will follow

## Context rule

Graphify output is navigation only.
Exact source files and tests are truth.

## Invocation rule

Do not run Graphify for every prompt.
Run it only when relationship discovery is useful.

## Generated output policy

Ignore generated graph output by default unless the project intentionally commits a small reviewed summary.
```

Do not hardcode local install paths.
Do not make Graphify a required dependency of the template.

### 4. Update `.agents/tools/repomix.md`

Content should say:

```md
# Repomix

## Role

Repomix is an optional portable context snapshot exporter.

## Use when

- sending controlled repo context to another AI tool
- creating an offline review artifact
- preparing a compact handoff snapshot
- sharing selected project files without giving direct repo access

## Do not use when

- working on a small known-file change
- targeted search and exact file reads are enough
- the snapshot would include secrets, dependencies, build outputs, or generated junk
- the snapshot is larger than the useful context needed

## Context rule

Repomix output is a snapshot, not truth.
It may be stale immediately after source changes.

## Invocation rule

Do not use Repomix as default coding context.
Use it only for controlled export/handoff.
```

## Expected final workflow description

After your changes, the repository should communicate this workflow:

```txt
For most prompts:
1. Read AGENTS.md and the relevant routing/context docs.
2. Use targeted search or exact known files.
3. Read source and tests before editing.
4. Make the smallest safe change.
5. Verify with lint/test/checks.

Use Aider:
- only when the developer is coding inside Aider.

Use Graphify:
- only when relationship/call-path/dependency discovery is needed.

Use Repomix:
- only when exporting a controlled snapshot to another AI/tool.

Use Understand Anything:
- only for broad onboarding or visual exploration of large/unfamiliar repos.

Use Caveman:
- only as terse-output style, not as architecture or analysis replacement.
```

## Acceptance criteria

Implementation is complete when:

1. No tracked tool documentation contains user-specific local machine paths.
2. Aider is documented as a coding executor, not only a repo map.
3. `.aider.conf.yml` exists with generic placeholders.
4. `.aiderignore` exists and excludes generated, secret, dependency, build, and media files.
5. `repomix.config.json` exists with safe ignores.
6. `.gitignore` exists and ignores generated outputs from these tools.
7. `context-strategy.md` clearly states that tools are escalation layers, not automatic prompt-time dependencies.
8. Graphify, Repomix, and Understand Anything are not recommended for every prompt.
9. Caveman is documented only as optional terse-output style.
10. Exact source files and tests are stated as the source of truth.
11. No unrelated agent or skill refactor is performed.
