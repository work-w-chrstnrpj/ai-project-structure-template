# Aider

## Role

Aider is an optional terminal coding executor.

## Use when

- the developer chooses Aider for implementation
- making focused code edits
- refactoring with Git-aware review
- using repo map as lightweight coding context

## Do not use when

- the task is documentation-only and simple
- the task only needs a quick answer
- the project owner did not choose Aider as the execution environment

## Context rule

Aider repo map is helpful orientation, not truth.
Before editing, read exact source files and relevant tests.

## Invocation rule

Aider is not automatically invoked by every prompt.
It is used only inside an Aider coding session.
