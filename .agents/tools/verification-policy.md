# Verification Policy

## Core Rule

Never claim verification passed unless it actually ran.

## Execution

- Use the smallest relevant verification command for the change.
- If project commands are placeholders, report that they need project-specific replacement.
- For template repo changes, verify with file structure checks, script execution, and text search.
- Report skipped verification with reason.

## Reporting

- Include the command run and its result when claiming success.
- Distinguish between recommended verification and completed verification.
