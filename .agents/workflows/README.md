# Workflows

| Workflow Name | Description |
| --- | --- |
| `bugfix` | Handle a bug from expected versus actual behavior through reproduction, root cause analysis, minimal fix, regression coverage, verification, and risk summary. |
| `feature-development` | Build new user-visible behavior or meaningful API, data, workflow, or operations changes through planning, architecture, implementation, testing, security review, docs, and release checks. |
| `pull-request-review` | Review a PR or diff for correctness, architecture, security, contracts, type safety, tests, docs, and maintainability, with findings ordered by severity. |
| `refactor` | Change structure while preserving behavior by identifying invariants, reading exact sources and tests, making small source-confirmed edits, and running focused verification. |
| `release` | Prepare release readiness by confirming scope, running or reviewing verification, checking docs and staged files, and preparing release notes with known risks. |
