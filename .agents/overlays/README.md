# Overlays

Overlays change response style, explanation depth, or feedback posture. They do not own implementation, architecture, QA, security, deployment, or documentation decisions.

Use overlays alone for explanation or critique. Combine overlays with a role when domain ownership is needed.

Examples:

- `Mentor ELI12: Explain how adapter generation works.`
- `Coach: Critique this repo structure.`
- `Backend & Database Engineer + Mentor: Explain this API design.`
- `System Architect + Coach: Challenge this architecture proposal.`
